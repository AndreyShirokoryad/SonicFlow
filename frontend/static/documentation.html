PlaylistAnalyze

PlaylistAnalyze помогает продолжить пользовательский плейлист: пользователь собирает список seed-треков, а сервис возвращает top-N треков, которые статистически подходят к этому плейлисту.

Функциональность сайта

  • Поиск треков по названию и артисту в словаре MPD-модели.
  • Добавление, удаление и изменение порядка seed-треков.
  • Импорт плейлиста из TXT или CSV.
  • Получение top-N рекомендаций с компонентами score.
  • Добавление рекомендованных треков обратно в плейлист.
  • Переход к треку в Spotify, если у результата есть Spotify URI.

На чем основан алгоритм

Первая версия использует Spotify Million Playlist Dataset. В обучении Item2Vec один плейлист считается предложением, а один трек считается словом. Если треки часто встречаются в похожих playlist-context, их embedding-векторы становятся близкими.

Модель не анализирует аудио напрямую. Основной сигнал - co-occurrence треков в реальных плейлистах Spotify. Для ускорения выдачи рекомендаций backend строит FAISS index по нормализованным Item2Vec-векторам и через него быстро находит candidate pool ближайших треков.

Формула ранжирования

score =
  0.45 * recent_similarity
+ 0.25 * whole_playlist_similarity
+ 0.20 * multi_seed_support
+ 0.10 * popularity_prior
+ preset_artist_weight * favorite_artist_affinity
  • recent_similarity - близость к верхним, самым свежим трекам плейлиста.
  • whole_playlist_similarity - близость к среднему вектору всего плейлиста.
  • multi_seed_support - поддержка несколькими seed-треками.
  • popularity_prior - мягкая поправка на популярность трека в MPD.
  • favorite_artist_affinity - бонус трекам артистов, которые часто встречаются в seed-плейлисте.

Режимы рекомендаций

  • Balance - сбалансированный режим по верхним свежим трекам, всему плейлисту, поддержке seed-треками и популярности.
  • More recent - сильнее продолжает верхние распознанные треки.
  • Favorite - сильнее учитывает любимых артистов плейлиста и поднимает их треки даже при небольшом MPD count.
  • More popular - сильнее поднимает треки, часто встречающиеся в MPD.

Основные API endpoints

  • GET /health - статус приложения и загруженной модели.
  • GET /model - параметры модели и служебная информация.
  • GET /tracks/search?q=... - поиск треков.
  • POST /tracks/batch - batch lookup треков по Spotify URI.
  • POST /recommend/match - подробный матчинг artist/title в MPD.
  • POST /recommend/playlist - рекомендации по текстовому плейлисту.
  • POST /recommend/uris - рекомендации по списку Spotify URI.

Production deployment

Проект подготовлен к запуску через Docker Compose. FastAPI отдает API и frontend, а Caddy проксирует приложение наружу и автоматически выпускает HTTPS-сертификат для публичного домена.
